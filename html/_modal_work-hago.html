📌 프로젝트 개요

리뉴얼을 통해 UX를 개선하고 성능을 최적화한 프로젝트입니다.
매출 성장에 기여할 수 있는 UI 구조와 성능 중심의 개선 작업이 핵심이었습니다.

🛠 주요 역할 및 기여
  • UI 리뉴얼: 메인 및 서브 페이지의 레이아웃 재설계로 사용자 경험 개선
  • 성능 최적화: 코드 최적화 및 불필요한 렌더링 제거를 통해 페이지 로딩 속도 30% 향상
  • 컴포넌트 표준화: 공통 UI 컴포넌트를 재사용 가능하도록 설계 및 모듈화
  • 매출 기여: 리뉴얼 이후 일매출 3천만 → 3억으로 성장한 프로젝트에 참여
🧩 전체 레이아웃 & UI 구조
  • Main - Mobile
    GNB
    스크롤 시 상단 플로팅 되도록 개발 CSS overflow를 사용하여 가로 스크롤 기능 추가하기
    레이아웃
    시맨틱 태그를 사용하여 header, footer를 작업하고 사용자의 스크롤 방향에 따라 header를 보여주고 감추는 기능 추가
    카테고리 메뉴
    카테고리 메뉴는 팝업창으로 열리며, 내부 메뉴는 accordion으로 열림/닫힘 기능 구현
    사이드 플로팅 버튼
    사용자의 스크롤 방향에 따라 top 버튼이 숨김/노출되고, 그에 맞춰 CSS로 상단 카카오 채널 버튼의 bottom 값 조절
    토스트 팝업
    타이머 영역 클릭 시 부모 영역에 on 클래스가 추가되며, translate로 열림/닫힘 구현하고 transition으로 모션 추가
  • Product List

    여러 페이지에 사용되는 카드형 및 슬라이드형 상품 리스트는 공통 UI 컴포넌트로 제작하고, 그 외 개별 페이지에서 쓰이는 상품 리스트는 Carousel 기능을 활용하여 작업하였습니다.

  • Main - Web
    GNB
    gsap를 활용하여 열림/닫힘 기능이 구현되며, mouseenter/mouseleave 이벤트에 따라 메뉴가 유지되거나 닫힘
    Header
    스크롤 이벤트를 감지하여 header 크기 이상으로 스크롤링이 발생하면 header 영역에 class를 추가시키고, 해당 class에 단축형 디자인에 맞는 position: fixed 등의 CSS를 추가하여 디자인 수정 및 위치 고정
    검색창
    사용자의 마우스와 키보드 이벤트를 감지하여 검색창 열림/닫힘과 방향키로 검색어를 제어할 수 있는 기능 추가
    사이드 플로팅 버튼
    사용자의 스크롤 이벤트 방향에 따라 노출/숨김
  • Product List

    Mobile과 마찬가지로 카드형 및 슬라이드형 상품 리스트는 공통 UI 컴포넌트로 제작하고, 그 외 개별 페이지에서 쓰이는 상품 리스트는 Carousel 기능을 활용하여 작업하였습니다.
